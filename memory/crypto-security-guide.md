# 加密货币安全指南 - 2026-02-07

## 核心安全原则

### 🔒 永远遵守

1. **Never Share Private Keys** - 永远不要分享私钥
2. **Hardware Wallet for Large Amounts** - 大额资金使用硬件钱包
3. **Enable 2FA** - 启用双重认证
4. **Verify URLs** - 验证网站 URL
5. **Avoid Public WiFi** - 避免使用公共 WiFi

## 诈骗识别

### 常见诈骗类型

#### 1. 虚假空投

**特征：**
- 承诺免费代币
- 要求连接钱包
- 要求支付小额 Gas 费用

**识别方法：**
- ✅ 查看官方公告
- ✅ 验证合约地址
- ✅ 不要信任不知名网站

#### 2. 钓鱼网站

**特征：**
- URL 与官方相似但不完全相同
- 看起来和官方网站一模一样
- 域名拼写错误

**识别方法：**
- ✅ 仔细检查 URL
- ✅ 使用书签保存官方地址
- ✅ 不要点击可疑链接

#### 3. 虚假客服

**特征：**
- 在 Discord/Telegram 假冒官方客服
- 要求转账"验证"
- 声称账户有问题

**识别方法：**
- ✅ 官方客服永远不会要求转账
- ✅ 验证客服 ID
- ✅ 直接联系官方支持

#### 4. 假冒项目

**特征：**
- 假冒知名项目
- 使用相似的 logo 和名称
- 发布虚假代币

**识别方法：**
- ✅ 验证官方社交媒体账户
- ✅ 检查合约地址
- ✅ 不要相信陌生人推荐

#### 5. 诈骗合约

**特征：**
- 伪装成正常 DEX
- 代币被错误识别
- 无法出售代币

**识别方法：**
- ✅ 验证合约代码
- ✅ 使用信誉良好的 DEX
- ✅ 不要批准未知的授权交易

## 智能合约安全

### 风险类型

#### 1. 重入攻击 (Reentrancy)

**描述：**
攻击者利用合约在更新状态前调用外部函数，反复提取资金。

**预防：**
- 使用 Checks-Effects-Interactions 模式
- 更新状态在转账之前

#### 2. 整数溢出

**描述：**
数字超过最大值，溢出到极小值或极大值。

**预防：**
- 使用 Solidity 0.8.0+ (内置溢出检查)
- 使用 SafeMath 库

#### 3. 访问控制问题

**描述：**
未授权的函数可以被调用，管理员权限管理不当。

**预防：**
- 使用 OpenZeppelin AccessControl
- 严格的权限检查

#### 4. 预言机操纵

**描述：**
攻击者操纵价格数据，从借贷协议获取优势。

**预防：**
- 使用多个价格源
- 使用时间加权平均价格 (TWAP)
- 使用信誉良好的预言机 (Chainlink)

## 交易安全

### 交易前检查清单

在执行任何交易前：

1. **🔍 验证网站 URL**
   - 确保是官方网站
   - 检查 SSL 证书
   - 避免使用搜索引擎直接搜索

2. **📝 验证合约地址**
   - 与官方公告对比
   - 在 Etherscan/BscScan 上验证
   - 检查合约是否已验证

3. **👛️ 检查代币信息**
   - 确认代币符号和名称
   - 检查总供应量
   - 查看智能合约代码

4. **💰 确认金额和地址**
   - 仔细检查转账金额
   - 验证接收地址
   - 不要从地址复制粘贴错误

5. **⏰ 等待确认**
   - 查看交易在浏览器上的状态
   - 等待足够确认数
   - 保存交易哈希

## DeFi 安全实践

### 智能合约交互

1. **永远不要批准无限额度**
   - 设置具体金额
   - 定期撤销授权

2. **使用信誉良好的协议**
   - Aave, Compound, Uniswap 等
   - 查看 DeFi 安全评分
   - 阅读安全审计报告

3. **了解无常损失**
   - AMM 流动性提供者会承担价格风险
   - 单边提供会遭受损失
   - 研究减少无常损失策略

4. **小额测试**
   - 先用小额测试
   - 确认流程正常后增加金额

### 流动性管理

1. **不要将所有资金存入一个池子**
   - 分散到多个池子
   - 分散到多个协议

2. **监控池子健康**
   - 查看流动性深度
   - 检查交易量
   - 注意异常活动

3. **及时撤出**
   - 如果发现可疑活动立即撤出
   - 如果协议被攻击立即撤出

## 应急响应

### 被黑或资金损失

**立即行动：**
1. **停止所有操作** - 不要进一步转账
2. **记录信息** - 保存交易哈希、地址、时间
3. **断开设备** - 断开可能受影响的设备
4. **更换私钥** - 如果私钥可能泄露，转移到新钱包
5. **报告事件** - 向相关交易所/平台报告

**调查：**
- 检查交易历史
- 查看授权记录
- 扫描设备是否有恶意软件
