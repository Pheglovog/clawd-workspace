### 研究内容

#### 项目概述
- **Chrome DevTools MCP**: Model Context Protocol (MCP) 服务器
- **功能**: 让 AI 编码助手（Claude, Cursor, Copilot）控制实时 Chrome 浏览器
- **维护者**: Chrome DevTools Team (Google)
- **工具数量**: 26 个工具

#### 核心功能

1. **输入自动化** (8 个工具)
   - click, drag, fill, fill_form, handle_dialog, hover, press_key, upload_file

2. **导航自动化** (6 个工具)
   - navigate_page, new_page, close_page, select_page, list_pages, wait_for

3. **仿真功能** (2 个工具)
   - emulate (设备仿真), resize_page (窗口大小调整)

4. **性能分析** (3 个工具)
   - performance_start_trace, performance_stop_trace, performance_analyze_insight

5. **网络分析** (2 个工具)
   - list_network_requests, get_network_request

6. **调试功能** (5 个工具)
   - evaluate_script, get_console_message, list_console_messages, take_screenshot, take_snapshot

#### MCP (Model Context Protocol)

MCP 是标准化 AI 助手与外部工具交互的协议：

```
AI 助手 (Claude/Cursor) 
    ↓ MCP 协议
MCP 服务器 (chrome-devtools-mcp)
    ↓ Chrome DevTools 协议
Chrome 浏览器
```

#### 应用场景

1. **自动化测试**
   - E2E 测试自动化
   - AI 自动识别元素
   - 无需编写测试代码
   - 节省 65-85% 测试成本

2. **性能优化**

3. **网络调试**
   - API 调试
   - 请求头分析
   - 响应时间分析
   - 快速定位问题

4. **爬虫和数据提取**
   - AI 理解页面结构
   - 自动处理反爬虫
   - 规避检测
   - 数据即服务

### 技术要点

#### 配置
```json
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": ["-y", "chrome-devtools-mcp@latest"]
    }
  }
}
```

#### 首次使用
```
提示: Check performance of https://developers.chrome.com
MCP 客户端会自动打开浏览器并记录性能追踪
```

#### 关键特性
- Puppeteer 自动等待（元素出现、网络空闲）
- Chrome DevTools 完整访问
- 多种连接方式（自动启动、连接已运行 Chrome、WebSocket）
- 源映射的堆栈跟踪
